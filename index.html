Why?


Separate the
Content / Markup / HTML
from the
Presentation / Layout / Styling

  1. So yeah, why css grid.
  2. we're trying to separate the content from the styling
  3. which was One of the ideals of the CSS Working Group
  4. that is, you try not to clutter the markup or the HTML itself with the layout description
  5. When we use frameworks like bootstrap we see our markup full of style description like col-md-6 and so on.
  6. Both flexbox and grid enable us to separate content from layout.
Both Flexbox and CSS Grid enable us to properly separate content from layout

Difference?

Flexbox is essentially for laying out items in a single dimension - in a row OR a column. Grid is for layout of items in two dimensions - rows AND columns.

Caveat

  1. CSS Grid and flexbox can be applied to just parts of the application
  2. No obligation to use it everywhere
  1. The caveat here is that if you use CSS Grid,
  2. it doesn't mean you have to write your entire application in CSS grid
  3. If we use a framework like bootstrap, we're essentially adopting it for the entire application
  4. But not so for flexbox and css grid, we can pick parts of the application where it makes sense, and use a mix of layout methods
  5. No obligation to use grid everywhere

grid-track


grid-template-columns: <column-track-size> <column-track-size>

  1. A grid-track is the space between any two adjacent grid-lines
  2. So with grid-template-columns, what we're defining is the track size of the different columns
  3. The highlighted bit is the first row-track

Caveat

Source order and accessibility

  1. Only visual reordering, not logical reordering
  2. Screen reader = source order
  3. Tab order = source order
  1. reordering things with grid layout or flexbox is only visual reordering
  2. not logical reordering
  3. Someone tabbing through the page could go wherever
